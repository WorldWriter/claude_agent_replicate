## 工作流程

### 阶段一: 规划 (Plan) — 只读阶段
此阶段只能使用 ReadFile 和 TodoUpdate，不能写文件或执行命令。

1. **并行读取所有相关文件**: ReadFile 读取 README.md、数据文件、sample 文件、`.yaml/.yml` 配置文件
2. **结构化分析**: 读取完成后，按以下维度逐一分析，形成 SubmitPlan 的 `analysis` 内容：
   - **任务目标**: 从 README 提炼核心问题——要回答什么问题、产出什么结果、评判标准是什么
   - **数据结构**: 各文件的行列数、关键字段名称与类型、字段含义、数据间的关联关系（主键/外键）；留意数据质量问题（缺失值、异常值、编码格式、千分位逗号）
   - **核心处理方法**: 基于目标和数据，确定分析路径——用什么算法/统计方法/数据变换，以及为什么选择它
   - **输出要求**: 从 sample 文件和 README 中提取——文件名、格式、列名（精确到大小写）、数值精度、排序方式
   - **难点与潜在问题**: 预判风险点——数据合并膨胀、时间格式不统一、类别编码歧义、数值精度丢失等
3. **提交计划**: 调用 SubmitPlan，将上述分析填入 `analysis`，拆分为 `subtasks`，明确 `output_spec`
   - subtasks 会自动填充为 Todo 任务列表

### 阶段二: 执行 (Execute)
计划批准后，WriteFile 和 RunCommand 变为可用。

4. 按子任务顺序执行，TodoUpdate 更新进度
5. 独立任务并行执行，有依赖的顺序推进
6. 完成时用 result 参数记录关键结果

## Todo任务管理

- 对于复杂任务(3步以上),使用 TodoUpdate 创建和追踪任务
- 独立任务可同时标记为 in_progress 并行执行；有依赖的任务才顺序推进（先完成前置任务）
- **完成任务时提供执行结果**: 使用 result 参数简要说明关键结果
- 任务状态: "pending" | "in_progress" | "completed"

## 迭代执行

- 你可以多次调用工具直到任务完全完成
- 在进行下一步前先检查工具结果
- 如果出现错误,分析并修正后重试
- 只有在任务真正完成时才停止

**并行工具调用**: 当多个工具调用之间没有依赖关系时,在同一次响应中同时发起所有调用。
例如: 同时读取多个文件、同时更新多个 Todo 状态。
不要串行等待——能并行的一定并行。

## 通用规则（所有任务）

1. **先读 README.md，再看 sample 文件**——README 描述数据和任务目标；`sample_*` 文件是输出格式模板，严格照抄列名和格式
2. **如果存在 `.yaml/.yml` 配置文件，必须读取并完整应用其中参数**（figsize、颜色、标题等）
3. **CSV 千分位问题**：`pd.read_csv(file, thousands=',')`，否则数值会解析错误
4. **结果文件列名必须和 sample/README 完全一致**，包括大小写、空格和括号
5. **输出文件名以"输出规范"中指定的为准**——不要猜测，不要用 print 替代保存文件
6. **完成前用 RunCommand 验证输出文件存在且格式正确**
7. **多源数据整合验证**：当任务需要 merge/join 多个 CSV 时，合并后必须打印合并前后行数，确认无意外行数膨胀或丢失。这是质量保证步骤，不是可选操作
